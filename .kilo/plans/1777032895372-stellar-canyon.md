# Comprehensive Fix Plan: Antibody Archive, Offense, & Sandbox

## Executive Summary

**Current State:** Only the Dashboard works reliably. Antibody Archive is empty, Offense strategies don't display properly, and Sandbox attacks don't persist to the archive. The user sees simulation activity but no stored results.

**Root Cause:** Silent failures in the data pipeline — antibody creation can fail without surfacing errors, breaking all downstream systems (archive queries return empty, offense never gets attack types to generate strategies, sandbox results don't persist).

**Fix Strategy:** Eliminate all silent failures, ensure every stage of the pipeline is robust, and verify the end-to-end data flow from sensor → detection → antibody → archive → UI. Add comprehensive error handling, logging, and data validation.

---

## Phase 1 — Critical Bug Fixes (BLOCKERS)

### 1.1 AntibodyStore._safe_json() — Stop Swallowing Serialization Errors

**Problem:** `_safe_json()` returns `'{}'` on ANY exception, converting non-serializable data (sets, bytes, custom objects) into empty objects. This corrupts `telemetry_json`, `response_json`, and `vector_json` fields, making later queries/display useless.

**Impact:** Telemetry data lost; archive modal shows empty raw telemetry; defensive insights incomplete.

**Fix:**

```python
@staticmethod
def _safe_json(obj) -> str | None:
    """Serialize obj to JSON. Returns None on failure (caller decides what to do)."""
    try:
        return json.dumps(obj, default=str)
    except Exception as e:
        logger.error(f'JSON serialization failed for {type(obj).__name__}: {e}')
        return None  # Don't mask failure with empty object
```

**Updates required:**
- `create()`: check each `_safe_json()` call; if any returns `None`, raise `ValueError('Failed to serialize antibody fields: telemetry/response/vector')`
- `update_insights()`: check for `None` before updating DB, raise if serialization fails

### 1.2 AntibodyStore.create() — Propagate DB Errors Instead of Silently Logging

**Problem:** Try/except around `self.db.execute()` and `self.db.commit()` catches all exceptions, logs them, but still returns the antibody dict (as if creation succeeded). Caller receives a truthy dict and believes antibody was persisted when it wasn't.

**Impact:** Antibodies never actually stored in database → archive queries empty, offense never sees attack types, strategy generation sees no new threats.

**Fix:**

```python
def create(self, threat: dict, response_taken: list, insights: dict = None) -> dict:
    antibody_id = str(uuid.uuid4())
    now = datetime.now(timezone.utc).isoformat()
    telemetry = threat.get('telemetry', {})
    mitre = threat.get('mitre_id', {})

    antibody = {
        'id': antibody_id, 'created_at': now,
        'attack_type': threat.get('attack_type') or 'unknown',
        'mitre_id': mitre.get('technique_id') if isinstance(mitre, dict) else None,
        'mitre_name': mitre.get('technique_name') if isinstance(mitre, dict) else None,
        'severity': int(threat.get('severity', 0)),
        'anomaly_score': float(threat.get('anomaly_score', 0.0)),
        'detection_ms': 0, 'neutralised_ms': 0,
        'source': telemetry.get('source', 'unknown'),
        'platform': platform.system(),
        'offensive_unlocked': 0,
    }

    # Serialize all fields with proper error handling
    telemetry_json = self._safe_json(telemetry)
    response_json = self._safe_json(response_taken)
    vector_json = self._safe_json(telemetry)

    if not all([telemetry_json, response_json, vector_json]):
        raise ValueError(
            f'Failed to serialize antibody fields: '
            f'telery={bool(telemetry_json)}, response={bool(response_json)}, vector={bool(vector_json)}'
        )

    antibody['telemetry_json'] = telemetry_json
    antibody['response_json'] = response_json
    antibody['vector_json'] = vector_json
    antibody['insights_json'] = self._safe_json(insights) if insights else None

    # Let DB errors propagate — caller will see exception
    self.db.execute("""INSERT INTO antibodies VALUES (
        :id, :created_at, :attack_type, :mitre_id, :mitre_name,
        :severity, :anomaly_score, :telemetry_json, :response_json,
        :vector_json, :detection_ms, :neutralised_ms, :source, :platform,
        :insights_json, :offensive_unlocked
    )""", antibody)
    self.db.commit()
    logger.info(f'Antibody created: {antibody_id} ({antibody["attack_type"]})')
    return antibody
```

**Remove:** the `try/except` block around execute/commit. Let exceptions bubble to `on_telemetry()`.

**Additional:** In `on_telemetry()`, wrap the `create()` call with try/except that:
  - Catches any exception from `create()` / DB operations
  - Logs the error (ERROR level)
  - Emits `ARCHIVE_ERROR` event: `emit('ARCHIVE_ERROR', {'message': str(e)})`
  - Sets `antibody = None` and returns early (so no strategy gen, no THREAT_NEUTRALISED)

### 2.2 Validate Attack Type Before Strategy Generation

**Problem:** `_ensure_strategy_for_attack_type()` uses `LIKE '%attack_type%'` which can match partial strings (e.g., `reverse_shell` matching `reverse_shell_ssh` if such a variant ever appears).

**Fix:** Implement exact CSV membership check by splitting stored `attack_types` and checking if `attack_type` is in the list. Use SQL string functions to avoid loading all rows in Python:

```python
def _ensure_strategy_for_attack_type(self, attack_type: str):
    if attack_type == 'unknown':
        return
    # Check if this attack_type already exists in any strategy's attack_types CSV
    # We match: 'attack_type' as a complete element (surrounded by commas or string boundaries)
    rows = self.db.execute("""
        SELECT id FROM offensive_strategies
        WHERE attack_types = ?
           OR attack_types LIKE ?
           OR attack_types LIKE ?
    """, (
        attack_type,
        f'{attack_type},%',    # at start: "ransomware,..."
        f'%%,{attack_type}%%', # in middle: "... ,ransomware,..."
    )).fetchall()
    # For end-of-string case (attack_type at end without trailing comma), use:
    # attack_types LIKE '%,attack_type' OR attack_types = 'attack_type'
    # The above covers all positions within comma-separated list
    if not rows:
        self.strategy_generator.create_strategy([attack_type])
```

**Alternative (cleaner):** Normalize storage (store JSON array instead of CSV). But CSV is already in use; the LIKE guard with prefix/suffix checks is safe.

**Decision:** Implement the LIKE-based exact-match guard now (low risk, high correctness).
