# Offense Module Implementation Plan

## Current State Analysis
- Offense page has a **global lock gate** requiring admin password for ALL access
- `offensive_strategies` table exists in DB with: id, created_at, name, description, attack_types, locked, unlock_key
- `StrategyGenerator` class exists (strategy_generator.py) with random name generation using ADJECTIVES/NOUNS
- `get_offensive_intel` aggregates antibody data — does **not** fetch strategies
- No IPC handlers exist for listing/unlocking individual strategies

## Desired State (from user request)
- List view of all offensive strategies with individual cards
- Each strategy has:
  - Random name generated by striker (mahoraga)
  - Individual lock icon on its card
  - Unlockable independently (not a global gate)
- Clicking a strategy (locked or unlocked) shows its details in a modal

## Implementation Steps

### 0. Database Migration (if needed)
- Confirm `offensive_strategies` table exists with correct schema (already present in db.py SCHEMA)
- Add migration if missing columns: name, description, attack_types, locked, unlock_key (already in SCHEMA)

### 1. Backend — IPC Handlers (main.py)
- Initialize `StrategyGenerator` in `MahoragaApp.__init__`: `self.strategy_generator = StrategyGenerator(self.db)`
- Add handler `GET_OFFENSIVE_STRATEGIES` → calls `self.get_offensive_strategies(payload)`
  - Query all strategies from DB (join with antibody stats for encounter counts)
  - Emit `OFFENSIVE_STRATEGIES_DATA` with list of strategies
- Add handler `UNLOCK_OFFENSIVE_STRATEGY` → calls `self.unlock_offensive_strategy(payload)`
  - Unlock a single strategy by ID with admin key (same hash check as global unlock)
  - Emit `OFFENSIVE_STRATEGY_UNLOCKED` with `{strategy_id, ok: boolean}`
- Optional: Keep existing `UNLOCK_OFFENSIVE` (global gate) but rename event to `OFFENSIVE_MODULE_UNLOCKED` for clarity

### 2. Backend — Auto-Generation Hook
- In `MahoregaApp.on_telemetry`, after `antibody = self.antibody_store.create(...)`, add:
  - `self._ensure_strategy_for_attack_type(attack_type)`
- New method:
  ```python
  def _ensure_strategy_for_attack_type(self, attack_type: str):
      # Check any existing strategy already covers this attack_type
      rows = self.db.execute(
          'SELECT id FROM offensive_strategies WHERE attack_types LIKE ?',
          (f'%{attack_type}%',)
      ).fetchall()
      if rows:
          return  # already covered
      self.strategy_generator.create_strategy([attack_type])
  ```
- StrategyGenerator.create_strategy already generates random name (ADJECTIVES/NOUNS) and inserts with `locked=1`

### 3. Frontend — offense.html Restructure
- Remove or make conditional the global gate overlay (id `off-gate`)
  - Add data attribute: `<body data-gate-mode="...">` or class to toggle visibility via JS
  - Gate becomes optional based on config/localStorage toggle
- Add page header with:
  - Title "Offensive Intelligence"
  - Stats: total strategies, locked strategies count
  - Toggle switch for "Require Admin Gate" (when toggled, store preference and show/hide gate overlay)
  - Global "Lock All" button
- Add filter bar: search input + filter tabs (All, Locked, Unlocked) — optional
- Replace technique grid with `<div class="off-strategy-grid" id="off-strategy-grid">` container
- Ready strategy detail modal (or reuse existing technique modal with modifications)

### 4. Frontend — offense.js Refactor
- Rename/restructure: `offense.js` now manages strategies not techniques
- State:
  - `_allStrategies = []`
  - `_currentFilter = 'all'`
  - `_gateMode = false` (default off)
  - `_gateUnlocked = false`
- `offInit()`:
  - Load gate mode preference (localStorage or config via IPC if available)
  - Register event handlers for: `OFFENSIVE_STRATEGIES_DATA`, `OFFENSIVE_STRATEGY_UNLOCKED`, `OFFENSIVE_MODULE_UNLOCKED` (if still present)
  - If gate mode ON and not unlocked: show gate overlay
  - Else: call `offLoadStrategies()`
- `offLoadStrategies()`: send `GET_OFFENSIVE_STRATEGIES`
- `_handleStrategiesData(data)`: store strategies, call `renderStrategies()`, update stats
- `renderStrategies()`: filter based on search/filter, then `off-strategy-grid.innerHTML = _allStrategies.map(buildStrategyCard).join('')`
- `buildStrategyCard(strategy)` returns HTML for card:
  - Strategy name (large)
  - Description (shortened)
  - Attack type badges (from strategy.attack_types CSV)
  - Count of techniques/encounters if available (join with antibody stats)
  - Lock state: if locked → show lock icon overlay, badge "LOCKED"; if unlocked → badge "UNLOCKED"
  - Click handler: `onclick="offShowStrategyModal(${JSON.stringify(JSON.stringify(strategy))})"`
  - For locked: also show small "Unlock" button on card; clicking it triggers unlock flow for that strategy
- `offShowStrategyModal(strategyJson)`: open modal with full strategy details
  - Show: name, description, attack types (as tags), creation date
  - If strategy.unlocked === 1: Show offensive context section — retrieve OFFENSIVE_CONTEXT for each attack_type and display combined or select one primary
  - If locked: Show lock icon, message "Admin access required", key input field, "Unlock" button
  - Unlock flow: send `UNLOCK_OFFENSIVE_STRATEGY` with `{strategy_id, key}`; on ok, update local strategy.locked=0 and re-render card; on fail, error toast
- Gate controls:
  - Toggle switch in header → sends IPC to update config (or localStorage) → triggers `offGateToggle(newMode)`
  - If toggled ON while unlocked, may show gate again; if toggled OFF, hide gate and load strategies
  - `offLockAll()`: mark all strategies as locked locally (not clearing server state) — just UI state
